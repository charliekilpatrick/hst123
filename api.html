API reference

Selected public modules are documented below. The main orchestration class hst123._pipeline.hst123 is large; only its summary is shown here—see the source or User guide for workflow details.

Entry point

hst123._pipeline.main()[source]

Entry point: parse args, download/copy data, align, drizzle, run dolphot, scrape.

Expects sys.argv[1] = RA, sys.argv[2] = Dec. Uses add_options for all other flags (–work-dir, –download, –run-dolphot, –scrape-dolphot, etc.). Exits on –help or invalid coord.

Utilities

Path helpers for FITS files (stable across os.chdir()).

The pipeline changes the working directory to --work-dir; relative paths must be normalized to absolute before that.

Context managers to silence stdout/stderr during noisy imports or C I/O.

Use suppress_stdout() for Python-level streams and suppress_stdout_fd() when C libraries write to file descriptor 1.

hst123.utils.stdio.suppress_stdout() → Iterator[None][source]

Redirect sys.stdout and sys.stderr to os.devnull until exit.

Yields:
None

Notes

Does not affect the C runtime stdout; see suppress_stdout_fd().

hst123.utils.stdio.suppress_stdout_fd() → Iterator[None][source]

Redirect OS file descriptor 1 (stdout) to os.devnull.

Yields:
None

Notes

For C extensions (e.g. DrizzlePac photeq) that write to the C stdio stdout. Python suppress_stdout() does not catch those writes. Stderr (fd 2) is left unchanged so logging handlers on stderr still work.

Parse DOLPHOT multi-extension output (catalog, .columns, .param, .data, .info, .warnings) and export a single HDF5 file with a labeled catalog plus metadata.

Designed against DOLPHOT 3.x text products: whitespace-separated numeric dpXXXX catalog, one descriptive line per column in dpXXXX.columns, and sidecar metadata files.

Primitives base

Base class for pipeline primitives (FITS, astrometry, DOLPHOT, etc.).

Each primitive receives the live hst123 instance as self._p and should use _primitive_cleanup() after writing outputs.
